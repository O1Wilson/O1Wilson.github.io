Challenges

Developing Fullstack01 presented several technical and learning challenges, especially in areas that were relatively new to me:

  • API Development in Python: One of the primary challenges was mastering API development in Python, where I implemented a robust RESTful architecture using Flask. I also integrated JWT (JSON Web Tokens) and OAuth for secure user authentication and authorization, ensuring that sensitive data remained protected throughout the application.
  • Firewall Configuration and TCP Rules: On the server-side, I had to handle firewall configuration and TCP rules, which were essential for protecting the server and controlling network traffic. I set up proper firewall rules to allow necessary traffic while preventing unauthorized access, and managed the security of TCP/IP connections to safeguard the network.
  • SQLAlchemy for Database Management: I worked extensively with SQLAlchemy, an ORM for Python, to manage complex database relationships and structures. This involved writing efficient queries, handling data migrations, and ensuring the integrity of the data across different stages of the project.
  • Apache2 and Flask with mod_wsgi: Configuring Apache2 with Flask using mod_wsgi was another key challenge in serving the application in a production environment. I configured the web server, optimized the setup for performance, and ensured a stable deployment of the Flask application.
  • Custom CSS and Keyframe Animations: I wrote hundreds of lines of custom CSS to ensure that the frontend was visually appealing and responsive. I implemented custom keyframe animations to enhance the user experience, adding smooth transitions and animations for interactive elements. This level of customization required me to balance performance with aesthetics, ensuring that the design was both functional and visually polished.

FullstackBG

This is a personal project of mine that I built for me and my friends. This website is designed for digital board gaming, with our own custom games. The website is built using Tailwind for the frontend, with future backend integration (Work in progress).

Challenges

Developing the Board Game Website as a solo project presented both time management and technical challenges. Balancing this project alongside other responsibilities made it difficult to dedicate consistent time, which taught me to:

  • Time Management & Project Ownership: Since this project was done in my personal time, balancing it alongside other responsibilities was difficult. I learned to plan sprints, set goals, and work effectively without a dedicated team or external deadlines. This project honed my ability to manage timelines independently and take full ownership of the work.
  • Node.js & Socket.io for Real-Time Features: This project marked my first significant hands-on experience with Node.js and Socket.io to handle real-time communication. Implementing WebSockets for multiplayer board games required me to:
    • Establish real-time connections between multiple players.
    • Handle event-driven programming efficiently and ensure smooth gameplay.
    • Optimize the performance of real-time features, including minimizing latency and ensuring that the server could handle multiple active connections concurrently.
  • Frontend-Backend Integration: Ensuring seamless integration between the Tailwind frontend and the Node.js backend required careful coordination. I focused on managing game state across sessions, synchronizing actions between players, and ensuring data persistence for ongoing games.
  • Testing & Debugging Real-Time Applications: Real-time features introduced complex scenarios that were not as straightforward to test. I learned how to implement robust logging, handle connection interruptions gracefully, and debug concurrency issues related to WebSockets.
  • Deployment & Maintenance: Deploying a live real-time application posed a new challenge. Configuring the server to manage concurrent connections efficiently and ensuring uptime were key focuses during deployment. Working on this project allowed me to bridge the gap between theoretical knowledge and practical application, particularly in building scalable real-time applications. I emerged more confident in my ability to work with modern JavaScript frameworks, manage project timelines independently, and deliver production-level software.
